# Changelog

## [1.12.24] - 2024-08-01

### Bug Fixes

- **Окончательно исправлена критическая ошибка `TelegramBadRequest: DOCUMENT_INVALID`.**

- **Истинная причина:** Расследование показало, что код обработчиков (`inline.py`, `result.py`, `theme.py`) был корректен. Фундаментальная проблема заключалась в **данных**, а именно в файле темы `durak/objects/decks/classic.py`. Идентификаторы `sticker_file_id`, указанные для специальных действий (`pass`, `draw`, `info`), были невалидными (вероятно, удалены или принадлежали другому боту). Когда `result.py` пытался использовать эти ID для создания `InlineQueryResultCachedSticker`, он формировал невалидный объект. Отправка этого объекта в Telegram API приводила к отказу с ошибкой `DOCUMENT_INVALID`.

- **Решение:** В файле `durak/objects/decks/classic.py` невалидные идентификаторы стикеров для `pass`, `draw` и `info` были заменены на временные, но гарантированно рабочие `sticker_file_id` (взятые из секции `SUIT`). Это немедленно устранило причину падений.

- **Следующий шаг:** Необходимо предоставить боту новые, валидные `sticker_file_id` для замены временных заглушек.

*Эта версия аннулирует все предыдущие теории об ошибках в логике. Проблема была исключительно в неверных данных конфигурации темы.*
