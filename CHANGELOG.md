## [1.7.5] - 2024-08-02

### Refactor
- **Внедрен единый экземпляр `GameManager` для управления состоянием игр:** Проведен глобальный рефакторинг с целью централизации управления игровыми сессиями. Ранее каждый файл-обработчик создавал свой собственный, изолированный экземпляр `GameManager`, что делало невозможным совместное использование данных о состоянии игр между различными командами (например, созданная в `new.py` игра была невидима для `join.py`).

        2.  **Внедрение зависимости (Dependency Injection):** Этот экземпляр передается в `Dispatcher` и становится доступным во всех обработчиках через аргументы функций.
        3.  **Удалены локальные экземпляры:** Из всех игровых обработчиков (`durak/handlers/game/*.py`) удалено создание локальных `gm = GameManager()`.
    - **Почему это важно:** Это критическое архитектурное исправление, которое решает фундаментальную проблему рассинхронизации состояния. Теперь все части бота работают с единым источником правды, что обеспечивает корректное взаимодействие между командами, стабильность игрового процесса и устраняет целый класс потенциальных ошибок, связанных с управлением состоянием.

- **Устранена критическая ошибка `AttributeError` и восстановлена система статистики:** Проведено комплексное исправление, которое устранило основную причину падений бота и восстановило функционал сбора статистики, который был сломан после миграции на Tortoise ORM.
    - **Что изменено:**
        1.  **Исправлены модели (`durak/db/models.py`):** 
            - В модель `ChatSetting` возвращено поле `is_game_active: bool`, которое необходимо для отслеживания активных игр. Его отсутствие приводило к `AttributeError` при создании или завершении игры.

            - В `ChatSetting` исправлен тип первичного ключа с `id` на `chat = fields.OneToOneField(\'models.Chat\', pk=True, related_name=\'settings\')`, что привело модель в соответствие с документацией `Tortoise ORM`.
        2.  **Обновлена игровая логика (`durak/logic/game_manager.py`, `durak/logic/actions.py`):**
            - Весь код, взаимодействующий с `ChatSetting`, был обновлен для использования `chat_id=` вместо `id=` при выполнении запросов к БД.
            - Восстановлена логика сбора статистики: в `game_manager.py` теперь обновляются счетчики побед и сыгранных игр в конце партии, а в `actions.py` — счетчики сыгранных, атаковавших и отбитых карт во время хода.
            - Добавлены защитные вызовы `User.get_or_create` и `Chat.get_or_create` для предотвращения ошибок целостности внешних ключей при сохранении статистики.
    - **Почему это важно:** Это исправление устраняет фундаментальную ошибку, которая вызывала "зависание" игр и "вечные" блокировки игроков из-за аварийного завершения функции `end_game`. Восстановление системы статистики возвращает важный соревновательный элемент игры и подготавливает кодовую базу к дальнейшему документированию и развитию.
