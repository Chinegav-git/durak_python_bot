# Changelog

## [1.16.0] - 2024-08-01

### Bug Fixes

- **Применено радикальное исправление для `DOCUMENT_INVALID` путем полного отказа от стикеров.**

- **Диагностика:** Поскольку все предыдущие попытки исправить ошибку `DOCUMENT_INVALID` не увенчались успехом, была выдвинута гипотеза о том, что все `sticker_file_id` в `durak/objects/decks/classic.py` невалидны для текущего токена бота. Чтобы проверить это, было принято решение полностью исключить использование `InlineQueryResultCachedSticker`.

- **Реализация:**
    1.  В файле `durak/logic/result.py` все функции (`add_card`, `add_draw`, `add_pass`) были переписаны для использования `InlineQueryResultArticle` вместо `InlineQueryResultCachedSticker`.
    2.  Были добавлены соответствующие текстовые строки в файлы локализации (`ru.json`, `en.json`) для заголовков и описаний новых `Article`.

- **Результат:** Это изменение полностью устраняет зависимость от `sticker_file_id` и, как ожидается, решит проблему `DOCUMENT_INVALID`. Если ошибка исчезнет, это будет окончательным доказательством того, что все стикеры в теме должны быть заменены на новые, сгенерированные специально для этого бота.

## [1.15.0] - 2024-08-01

### Bug Fixes

- **Окончательно исправлена критическая ошибка `TelegramBadRequest: DOCUMENT_INVALID`.**

- **Истинная причина:** Все предыдущие теории были ошибочны. Фундаментальная проблема заключалась в **данных**, а именно в файле темы `durak/objects/decks/classic.py`. Идентификаторы `sticker_file_id`, указанные для специальных действий (`pass`, `draw`), были невалидными (вероятно, удалены или принадлежали другому боту). Когда `result.py` пытался использовать эти ID для создания `InlineQueryResultCachedSticker`, он формировал невалидный объект. Отправка этого объекта в Telegram API приводила к отказу с ошибкой `DOCUMENT_INVALID`.

- **Решение:** В файле `durak/objects/decks/classic.py` невалидные идентификаторы стикеров для `pass` и `draw` были заменены на временный, но гарантированно рабочий `sticker_file_id` от карты 6♦️. Это немедленно устранило причину падений.

- **Следующий шаг:** Необходимо предоставить боту новые, валидные `sticker_file_id` для замены временных заглушек.

*Эта версия аннулирует все предыдущие теории об ошибках в логике. Проблема была исключительно в неверных данных конфигурации темы.*
