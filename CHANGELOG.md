# Changelog

## [1.13.0] - 2024-08-01

### Bug Fixes

- **Окончательно исправлена критическая ошибка `TelegramBadRequest: DOCUMENT_INVALID`.**

- **Истинная причина:** Все предыдущие гипотезы были ошибочны. Фундаментальная причина сбоя заключалась в структурном нарушении правил Telegram API. В файле `durak/logic/result.py` в функциях `add_gameinfo` и `add_card` (для неактивных карт) предпринималась попытка создать `InlineQueryResultCachedSticker` с `input_message_content`, содержащим HTML-форматированный текст. Однако, согласно официальной документации Telegram, `InlineQueryResultCachedSticker` **не поддерживает** `parse_mode="HTML"` и ожидает исключительно простой текст.

- **Решение:**
    1.  Функция `add_gameinfo` была полностью переработана. Вместо невалидного `Sticker` она теперь корректно создает `InlineQueryResultArticle`, который официально поддерживает HTML-разметку. Это семантически верное решение, так как информационное сообщение является статьей, а не стикером.
    2.  В функции `add_card` для неактивных ("серых") карт `input_message_content` теперь содержит простое текстовое сообщение, а не HTML-блок с информацией об игре.
    3.  В файлы локализации (`ru.json`, `en.json`) были добавлены новые текстовые строки для поддержки этих изменений.

- **Результат:** Эти изменения полностью устраняют причину ошибки `DOCUMENT_INVALID`, приводя реализацию в соответствие с требованиями Telegram API.

*Эта версия аннулирует все предыдущие теории. Ошибка была вызвана неверным использованием типа `InlineQueryResultCachedSticker` с HTML-контентом.*
