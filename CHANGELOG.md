## [1.9.0] - 2025-02-29

### 🚨 Critical Fixes
- **Полностью устранена ошибка `ModuleNotFoundError`, блокировавшая запуск бота:**
    - **Проблема:** После удаления устаревшего файла `durak/filters/is_admin.py` (в рамках исправления v1.8.8), в кодовой базе остались необработанные импорты этого модуля, что приводило к фатальной ошибке `ModuleNotFoundError` при старте.
    - **Причина неполного исправления:** Ассистент (Gemini) совершил **повторную ошибку**, не проведя исчерпывающий поиск по всей кодовой базе. Исправления вносились поэтапно (`settings.py`, `kick.py`, затем `kill.py`, и наконец `get_sticker_id.py`), что требовало нескольких перезапусков и демонстрировало некачественный анализ.
    - **Решение:**
        1.  Проведен полный аудит кодовой базы для поиска всех вхождений `durak.filters.is_admin`.
        2.  В файлах `kick.py`, `settings.py`, и `kill.py` вызовы старого фильтра заменены на корректные функции из `durak/logic/utils.py` для проверки прав администратора чата.
        3.  В файле `get_sticker_id.py` проверка заменена на `utils.is_bot_admin` для соответствия логике "только для глобальных админов".

### Code Quality
- **Унификация проверки прав:** Логика проверки прав доступа полностью централизована в модуле `durak/logic/utils.py`, что повышает надежность и упрощает дальнейшую поддержку.

## [1.8.9] - 2025-02-28

### 🐞 Bug Fixes
- **Исправлена ошибка `AttributeError` при запуске бота:**
    - **Проблема:** В файле `durak/handlers/game/actions.py` использовалась некорректная аннотация типа `types.Bot` вместо `Bot`. Поскольку класс `Bot` находится напрямую в `aiogram`, а не в `aiogram.types`, это приводило к ошибке атрибута и падению бота на старте.
    - **Решение:** В `actions.py` добавлен корректный импорт `from aiogram import Bot` и исправлена аннотация типа в сигнатуре функции `_handle_special_sticker`.
