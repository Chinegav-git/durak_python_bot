# Архитектура проекта / Project Architecture

## Обзор архитектуры / Architecture Overview

Проект представляет собой Telegram-бота для игры в "Дурак" с использованием современной асинхронной архитектуры на основе aiogram 3.x.
The project is a Telegram bot for playing "Durak" card game using modern asynchronous architecture based on aiogram 3.x.

## Основные компоненты / Core Components

### 1. Точка входа / Entry Point
- **`bot.py`** - Основной файл запуска бота
  - Инициализация Bot и Dispatcher
  - Настройка Redis storage для FSM
  - Создание единого экземпляра GameManager
  - Регистрация middleware и обработчиков
  - Управление жизненным циклом приложения

### 2. Система внедрения зависимостей / Dependency Injection System

#### GameManagerMiddleware
- **Файл:** `durak/middleware/game_manager_middleware.py`
- **Назначение:** Автоматическая передача GameManager во все обработчики
- **Принцип работы:** 
  - Перехватывает входящие сообщения и callback'и
  - Добавляет `gm` в словарь `data` обработчика
  - Обеспечивает единый доступ к игровому состоянию

### 3. Управление callback запросами / Callback Management System

#### CallbackManager
- **Файл:** `durak/utils/callback_manager.py`
- **Функциональность:**
  - Предотвращение дублирования callback'ов (5 минут)
  - Ограничение одновременных запросов (максимум 10)
  - Обработка таймаутов (30 секунд)
  - Автоматическое очищение старых записей
  - Безопасная обработка ошибок Telegram API

### 4. Игровой менеджер / Game Manager

#### GameManager
- **Файл:** `durak/logic/game_manager.py`
- **Ответственность:**
  - Управление игровыми сессиями
  - Хранение состояния игр в Redis
  - Координация действий игроков
  - Обработка игровой логики

### 5. Обработчики команд / Command Handlers

#### Структура:
```
durak/handlers/
├── game/           # Игровые команды
│   ├── new.py      # Создание игры
│   ├── join.py     # Присоединение
│   ├── start.py    # Начало игры
│   ├── actions.py  # Игровые действия
│   └── ...
├── info/           # Информационные команды
│   ├── help.py     # Помощь
│   ├── stats.py    # Статистика
│   └── ...
└── settings.py     # Настройки
```

### 6. Игровая логика / Game Logic

#### Основные компоненты:
- **`durak/logic/actions.py`** - Обработка игровых действий
- **`durak/logic/result.py`** - Определение результатов
- **`durak/logic/utils.py`** - Вспомогательные функции

### 7. Объекты игры / Game Objects

#### Модели данных:
- **`durak/objects/game.py`** - Игровая сессия
- **`durak/objects/player.py`** - Игрок
- **`durak/objects/card.py`** - Карта
- **`durak/objects/deck.py`** - Колода
- **`durak/objects/decks/`** - Типы колод

### 8. База данных / Database

#### Tortoise ORM
- **`durak/db/models.py`** - Модели данных
- **`durak/db/tortoise_config.py`** - Конфигурация
- **Таблицы:**
  - `Chat` - Информация о чатах
  - `User` - Пользователи
  - `Game` - Игровые сессии
  - `ChatSetting` - Настройки чатов
  - `UserSetting` - Настройки и статистика пользователей

### 9. Утилиты / Utilities

#### Вспомогательные компоненты:
- **`durak/utils/callback_decorator.py`** - Декораторы для callback
- **`durak/filters/`** - Фильтры сообщений
- **`durak/middleware/`** - Middleware компоненты

## Поток данных / Data Flow

### 1. Создание игры
```
Пользователь → /new → GameManagerMiddleware → new.py → GameManager → Redis
```

### 2. Присоединение к игре
```
Пользователь → "Присоединиться" → CallbackManager → join.py → GameManager → Redis
```

### 3. Игровое действие
```
Пользователь → Callback → CallbackManager → actions.py → GameManager → Обновление состояния
```

## Принципы архитектуры / Architectural Principles

### 1. Единый источник правды (Single Source of Truth)
- GameManager является центральным компонентом управления состоянием
- Redis используется для хранения игрового состояния
- PostgreSQL для персистентных данных

### 2. Внедрение зависимостей (Dependency Injection)
- Middleware система для передачи зависимостей
- Отсутствие глобальных переменных

### 3. Асинхронность (Asynchrony)
- Все операции ввода-вывода асинхронные
- Использование asyncio для параллельной обработки

### 4. Обработка ошибок (Error Handling)
- Централизованная обработка callback ошибок
- Логирование всех критических событий

### 5. Масштабируемость (Scalability)
- Модульная структура обработчиков
- Легкое добавление новых команд

## Безопасность / Security

### 1. Проверка прав доступа
- Фильтры администрирования
- Проверка создателей игры

### 2. Защита от злоупотреблений
- Ограничение одновременных callback'ов
- Предотвращение дублирования запросов

### 3. Валидация данных
- Проверка входных параметров
- Безопасная обработка пользовательского ввода

## Производительность / Performance

### 1. Кэширование
- Redis для игрового состояния
- Быстрый доступ к активным играм

### 2. Оптимизация запросов
- Эффективная работа с базой данных
- Минимизация блокирующих операций

### 3. Управление ресурсами
- Автоматическое очищение старых данных
- Ограничение одновременных операций
